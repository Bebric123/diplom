Подключение бота к чату

Connect the bot to a chat

Коллектор шлёт алерты в Telegram тем же ботом, чей токен указан в TELEGRAM_BOT_TOKEN. Чат, куда слать сообщения, задаётся при регистрации проекта (поле Telegram chat id).

The collector sends alerts with the bot from TELEGRAM_BOT_TOKEN. The destination chat is set at project registration (Telegram chat id).

1. Токен бота (BotFather)

1. Bot token (BotFather)

  1. В Telegram откройте @BotFather → /newbot → имя и username.
  2. Скопируйте токен вида 123456:ABC… в переменную TELEGRAM_BOT_TOKEN в .env на сервере (Docker: project/docker/.env).
  3. Перезапустите сервис bot (и при необходимости backend/worker), чтобы подтянуть конфиг.
  1. Open @BotFather → /newbot.
  2. Copy the token into TELEGRAM_BOT_TOKEN in your server .env (e.g. project/docker/.env).
  3. Restart the bot container (and stack if needed).

2. Группа или супергруппа для алертов

2. Group or supergroup for alerts

  1. Создайте группу в Telegram (лучше супергруппа — так проще с большими чатами).
  2. Добавьте в группу вашего бота (по username из BotFather).
  3. При желании сделайте бота администратором — так надёжнее доставляются длинные сообщения и вложения.
  1. Create a Telegram group (a supergroup is fine).
  2. Add your bot to the group.
  3. Optionally promote the bot to admin for more reliable delivery of long messages.

3. Узнать chat id

3. Get the chat id

Нужно число вида -1001234567890 (супергруппа) или -123456789 (группа). Варианты:

You need a numeric id like -1001234567890 (supergroup). Options:

  • Напишите в группе любое сообщение и откройте в браузере
    https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates
    найдите "chat":{"id": … — это и есть chat id.
  • Либо используйте ботов-помощников (например, добавьте в группу бота, который показывает id чата) — по правилам вашей организации.
  • Post any message in the group, then open
    https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates
    and read "chat":{"id": ….
  • Or use a helper bot allowed by your policy.

Токен в URL — секрет: не пересылайте ссылку третьим лицам и не коммитьте в git.

The token in the URL is a secret — do not share or commit it.

4. Регистрация на коллекторе

4. Register on the collector

  1. Откройте /register на коллекторе (например http://127.0.0.1:8000/register).
  2. Вставьте Telegram chat id без лишних пробелов (формула нормализации уберёт пробелы внутри числа).
  3. Сохраните project_id и API-ключ для SDK.
  1. Open /register on your collector host.
  2. Paste the Telegram chat id (spaces inside the number are stripped).
  3. Save project_id and the API key for the SDK.

5. Команды и кнопки в чате

5. Commands and buttons in chat

В этом же чате бот понимает проект и отвечает на:

In that same chat the bot resolves the project and handles:

  • /stats — сводка за 7 дней;
  • /report — Excel за 7 дней;
  • /task <uuid> — карточка задачи;
  • кнопки под алертами — «Взять в работу», «Решена».
  • /stats — 7-day summary;
  • /report — Excel for 7 days;
  • /task <uuid> — task card;
  • inline buttons on alerts — acknowledge / resolve.

В личных сообщениях боту команды /stats и /report не привязаны к проекту — используйте группу с алертами.

/stats and /report do not work in private chat — use the project group.

Если алерты не приходят

If alerts do not arrive

  • telegram_chat_id в БД должен совпадать с id группы (частая ошибка — другой чат или лишний пробел).
  • Токен в .env — от того же бота, который добавлен в группу.
  • Коллектор и SDK: MONITOR_URL должен указывать на FastAPI-коллектор, а не на другое приложение на том же порту.
  • Смотрите логи контейнеров worker и bot.
  • telegram_chat_id must match the group id.
  • The token must be for the same bot that is in the group.
  • MONITOR_URL must point to the collector API.
  • Check worker and bot logs.
