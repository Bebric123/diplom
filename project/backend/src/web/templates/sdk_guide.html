Введение

Overview

Есть два независимых способа работы с коллектором — их не нужно смешивать в одном «потоке», хотя в одном приложении можно использовать оба параллельно:

  1. События и ошибки — POST /track. События (track_event), ручной вызов capture_exception, middleware (FastAPI и т.д.) отправляют JSON о действии или исключении. Это не загрузка файла лога.
  2. Файлы логов — POST /logs/upload. Отдельный endpoint: вы передаёте текст лога (например последние N строк). В SDK это сейчас реализовано удобнее всего для Python (send_log_file); остальные стеки — через HTTP или свой код.

Авторизация везде одна: API-ключ из регистрации — Authorization: Bearer <ключ> или X-Api-Key (как настроено на сервере).

There are two separate integration paths (you can use both in one app, but they are different APIs):

  1. Events & errors — POST /track (track_event, capture_exception, framework middleware). Not log-file upload.
  2. Log files — POST /logs/upload. You send log text (e.g. last N lines). First-class helper today is Python send_log_file; other stacks use HTTP or custom code.

Auth everywhere: API key — Authorization: Bearer … or X-Api-Key.

Ключи и базовый URL

Keys & base URL

После регистрации сохраните project_id (UUID) и API-ключ. В коде SDK:

  • endpoint / базовый URL — без пути /track (например https://collector.example.com).
  • project_id / projectId — строка UUID проекта.
  • api_key / apiKey — секрет для заголовка.

After registration, store project_id and the API key. In SDKs:

  • endpoint — base URL without /track.
  • project_id / projectId — project UUID string.
  • api_key / apiKey — secret for headers.

Python

Сначала init_monitor один раз при старте. Используйте два разных канала по ситуации: события/ошибки — POST /track; хвост лог-файла — POST /logs/upload (раздел Логи). Это разные API, не один «универсальный» вызов.

capture_exception, track_event, middleware FastAPI — всё это только /track, не загрузка файла лога.

Call init_monitor once. Two channels: /track for events/exceptions; log tail → /logs/upload (see Logs). Different APIs.

События и автоматический отлов ошибок (/track)Events & automatic errors (/track)

from error_monitor_sdk import init_monitor, capture_exception, track_event

init_monitor(
    endpoint="https://YOUR_COLLECTOR",
    project_id="YOUR_PROJECT_UUID",
    api_key="YOUR_API_KEY",
)

track_event("payment_ok", {"amount": "99.00"})

try:
    risky()
except Exception as e:
    capture_exception(e, {"route": "/api/checkout"})

# FastAPI: подключите middleware из error_monitor_sdk.integrations.fastapi
# для автоматической отправки необработанных исключений

Отправка логов — последние N строк файла (/logs/upload)Log upload — last N lines (/logs/upload)

Отдельно от отлова исключений: функция send_log_file из error_monitor_sdk.logs читает файл с диска и шлёт последние lines строк на коллектор. Имеет смысл вызывать из обработчика ошибки (или по таймеру), например чтобы приложение само приложило хвост лога к инциденту.

Separate from /track: send_log_file in error_monitor_sdk.logs reads a file and uploads the last lines lines. Call from your error handler or on a schedule.

from error_monitor_sdk import init_monitor
from error_monitor_sdk.logs import send_log_file

init_monitor(
    endpoint="https://YOUR_COLLECTOR",
    project_id="YOUR_PROJECT_UUID",
    api_key="YOUR_API_KEY",
)

def on_critical_failure():
    send_log_file(
        "/var/log/myapp/app.log",
        lines=200,
        service_name="api",
        environment="production",
    )

PHP

Установка — Composer (path). Сейчас SDK покрывает события и исключения (/track): init_monitor, track_event, capture_exception. Отдельной функции «прочитать файл и отправить последние N строк» в PHP SDK пока нет — логи на коллектор можно слать сырым HTTP на /logs/upload.

Install via Composer (see above). The PHP SDK focuses on /track (events + exceptions). No built-in log-tail helper yet — use raw HTTP for /logs/upload.

<?php
require_once 'vendor/autoload.php';

use function ErrorMonitor\init_monitor;
use function ErrorMonitor\track_event;
use function ErrorMonitor\capture_exception;

init_monitor(
    'https://YOUR_COLLECTOR',
    'YOUR_PROJECT_UUID',
    null, // user id callable
    [],
    'YOUR_API_KEY'
);

track_event('order_paid', ['order_id' => '42']);

try {
    throw new RuntimeException('fail');
} catch (Throwable $e) {
    capture_exception($e, ['handler' => 'checkout']);
}

HTTP без SDK (curl)

Raw HTTP (curl)

События и ошибки — POST /track

JSON: project_id, action, timestamp (ISO 8601), context, meta. Для сохранения ошибки в БД добавьте в meta поля из раздела Поля ошибки.

Events — POST /track

Body: project_id, action, timestamp, context, meta. Error fields in meta: see below.

curl -sS -X POST "https://YOUR_COLLECTOR/track" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "project_id": "YOUR_PROJECT_UUID",
    "action": "manual_ping",
    "timestamp": "2025-01-01T12:00:00.000Z",
    "context": {
      "platform": "backend",
      "language": "curl",
      "os_family": "Linux",
      "browser_family": "server"
    },
    "meta": {
      "user_id": "batch",
      "page_url": "https://example.com/job"
    }
  }'

Браузер (fetch)

Browser (fetch)

С фронтенда вызывайте тот же /track. Учтите CORS: на коллекторе должен быть разрешён origin вашего сайта. Не вшивайте API-ключ в публичный бандл — для продакшена используйте прокси на своём бэкенде или отдельный публичный ключ с ограничениями, если такая политика у вас настроена.

Call the same /track from the browser. Configure CORS on the collector. Avoid embedding secrets in public bundles; prefer a backend proxy.

await fetch('https://YOUR_COLLECTOR/track', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    Authorization: 'Bearer YOUR_API_KEY',
  },
  body: JSON.stringify({
    project_id: 'YOUR_PROJECT_UUID',
    action: 'page_view',
    timestamp: new Date().toISOString(),
    context: {
      platform: 'browser',
      language: 'javascript',
      browser_family: navigator.userAgent,
    },
    meta: { page_url: location.href, user_id: 'anon' },
  }),
});

В репозитории SDK для JS есть интеграция integrations/browser.js для автоматической отправки.

See integrations/browser.js in sdk-js for automated capture.

Логи (файлы) — /logs/upload

Log files — /logs/upload

Это не /track: вы отправляете текст лога (часто последние строки файла), чтобы на стороне коллектора его проанализировать. В Python из приложения удобно вызвать send_log_file(путь, lines=N, …) — SDK сам прочитает хвост файла и соберёт JSON. В Node — sendLogFile после initMonitor. Типичный сценарий: при ошибке в обработчике дополнительно отправить последние N строк app.log.

Поля JSON: project_id, filename, content, lines_sent, опционально total_lines, server_name, service_name, environment, error_group_id.

Not /track: you send log text (often the tail of a file). Python: send_log_file(path, lines=N); Node: sendLogFile after initMonitor. Typical: on failure, attach the last N lines of app.log.

JSON: project_id, filename, content, lines_sent, optional total_lines, server_name, service_name, environment, error_group_id.

curl -sS -X POST "https://YOUR_COLLECTOR/logs/upload" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "project_id": "YOUR_PROJECT_UUID",
    "filename": "worker.log",
    "content": "line1\nline2\n",
    "lines_sent": 2,
    "total_lines": 2,
    "environment": "production",
    "service_name": "worker"
  }'

Поля ошибки в событии /track

Error fields in /track meta

Только для потока событий/ошибок (не для /logs/upload). Чтобы коллектор сохранил стек и сообщение как ошибку, в meta укажите: error_message, error_stack, при необходимости error_line, error_column, error_file. В SDK это делает capture_exception.

Set in meta: error_message, error_stack, optionally error_line, error_column, error_file. SDKs populate these in capture_exception.

"meta": {
  "user_id": "u1",
  "page_url": "https://app.example.com/checkout",
  "error_message": "Payment provider timeout",
  "error_stack": "Error: ...\n  at ...",
  "error_file": "payments.js",
  "error_line": 120
}
